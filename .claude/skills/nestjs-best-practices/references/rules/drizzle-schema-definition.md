## Intent

Define all database tables using `pgTable()` from `drizzle-orm/pg-core` and export `InferSelectModel`/`InferInsertModel` type aliases for type-safe service code.

## Why

Drizzle is a code-first ORM where the TypeScript schema IS the source of truth. Using `pgTable()` correctly ensures the compiler enforces column types throughout the codebase. Exporting `InferSelectModel` and `InferInsertModel` lets services and DTOs use accurate types without duplication. Deviating from this pattern (e.g. manual type definitions, raw SQL strings for schema) breaks the type-safety guarantee that makes Drizzle valuable.

## Apply When

- Creating any new database table
- Adding columns to an existing table
- Defining relationships between tables
- Creating indexes or constraints

## Do Not Apply When

- Writing migration SQL files directly (these are generated by `drizzle-kit generate`)
- Defining view types (use `pgView()` instead)

## Required Pattern

```typescript
// src/db/schema/users.schema.ts
import {
  pgTable,
  uuid,
  varchar,
  text,
  timestamp,
  boolean,
} from 'drizzle-orm/pg-core';
import type { InferSelectModel, InferInsertModel } from 'drizzle-orm';

export const users = pgTable('users', {
  id: uuid('id').primaryKey().defaultRandom(),
  email: varchar('email', { length: 255 }).notNull().unique(),
  name: varchar('name', { length: 255 }).notNull(),
  passwordHash: text('password_hash').notNull(),
  isActive: boolean('is_active').notNull().default(true),
  createdAt: timestamp('created_at').notNull().defaultNow(),
  updatedAt: timestamp('updated_at').notNull().defaultNow(),
});

// Always export these — used by services and DTOs
export type User = InferSelectModel<typeof users>;
export type NewUser = InferInsertModel<typeof users>;
```

## Forbidden Pattern

```typescript
// @ts-nocheck
// ❌ Manual type definition — duplicates schema, drifts from DB reality
interface User {
  id: string;
  email: string;
  // ... manually kept in sync
}

// ❌ Using TypeORM entities
@Entity('users')
class UserEntity {
  @PrimaryGeneratedColumn('uuid')
  id: string;
}

// ❌ Missing type exports — forces consumers to use `any` or duplicate types
export const users = pgTable('users', {
  id: uuid('id').primaryKey().defaultRandom(),
  email: varchar('email', { length: 255 }).notNull(),
});
// No exported types — bad!
```

## Review Checklist

- [ ] Table defined with `pgTable()` from `drizzle-orm/pg-core`
- [ ] All columns use correct Drizzle column builders (`uuid`, `varchar`, `text`, `timestamp`, `boolean`, `integer`, etc.)
- [ ] `InferSelectModel` type alias exported as the singular entity name (e.g. `User`)
- [ ] `InferInsertModel` type alias exported with `New` prefix (e.g. `NewUser`)
- [ ] Schema file is in `src/db/schema/` and re-exported from `src/db/schema/index.ts`
- [ ] Sensitive fields (passwordHash, etc.) are present in schema but excluded from response DTOs

## Stack-Specific Notes

- Import column builders from `drizzle-orm/pg-core` — not from `drizzle-orm` directly
- UUID primary keys: use `.defaultRandom()` for auto-generation
- Timestamps: use `.defaultNow()` for `createdAt`; `updatedAt` requires manual update or a DB trigger
- Column naming: use snake_case in DB (`password_hash`), camelCase in TypeScript (`passwordHash`)
- All schema files must be registered in the Drizzle Kit config (`drizzle.config.ts`) via the `schema` glob
